## Cross-Phase Integration (7/8 wired)

The full happy-path pipeline is automatic — no manual trigger between stages. Auth (AUTH-01)
confirmed protected: token only ever in `Process.environment` (env-first, then `gh auth token`),
never in `AppState` or the command string.

E2E chain traced and wired:
launcher JSON → bind repo → hotkey → record `latest.wav` (16k mono PCM) → bundled `whisper-cli` →
transcript → `IssueFilingRunner.file(transcript, repo)` with `cwd = repo.rootURL` → `claude -p` files
via scoped GitHub MCP (`--allowedTools mcp__github__issue_write`) → `IssueResultParser` extracts
`#N` from the `/issues/N` URL → `speak("created issue #N")`.

- **FINDING-06 (WARNING / PROVIDER-01):** the "CLI Command" UI field writes `cliCommandKey` to
  UserDefaults, but no production path reads it — `onRunIssueFiling` hardcodes `.claudeGitHub`
  (`cliCommand: "claude"`). The field is a false affordance. Not a blocker (the validated leg works
  and the config seam is provider-agnostic in code), but the control should be hidden or wired
  before non-Claude providers are offered.
