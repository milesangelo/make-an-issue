<threat_model>

## Trust Boundaries

| Boundary | Description |
| -------- | ----------- |
| Build script -> app bundle | Local script copies executable and plist into an openable app bundle. |
| App bundle metadata -> macOS LaunchServices | `Info.plist` controls how macOS launches and displays the app. |
| App state -> menu UI | Internal state becomes user-visible menu content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --------- | -------- | --------- | ----------- | --------------- |
| T-01-01-S | Spoofing | `Resources/Info.plist` bundle identifier | mitigate | Use one explicit bundle identifier, assert it with `plutil`, and have later launcher work target this bundle or generated app path. |
| T-01-01-T | Tampering | `scripts/build-app.sh` app bundle assembly | mitigate | Script must derive paths from the repository root, quote filesystem paths, create only `.build/MakeAnIssue.app`, and never accept arbitrary copy destinations. |
| T-01-01-I | Information Disclosure | `MenuView` initial state | mitigate | Render only app status and repo display text from `AppState`; do not expose environment variables, shell output, or filesystem listings. |
| T-01-01-D | Denial of Service | SwiftUI app startup | mitigate | Keep app startup limited to state creation and menu rendering; defer launch handoff and repo resolution to later plans with tests. |
| T-01-01-E | Elevation of Privilege | Non-sandboxed app setup | mitigate | Do not add external command execution in this plan; Phase 1 app bundle is non-sandboxed only to support later local CLI phases and starts with no privileged operations. |

</threat_model>
