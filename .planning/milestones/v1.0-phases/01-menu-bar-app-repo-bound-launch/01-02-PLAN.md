<threat_model>

## Trust Boundaries

| Boundary | Description |
| -------- | ----------- |
| User shell cwd -> launcher script | The current directory is untrusted input captured by `bin/make-an-issue`. |
| Launcher script -> Application Support JSON | The launcher serializes cwd data for the GUI app. |
| Application Support JSON -> running app | The app reads a local file that may be modified by the user account. |
| Launcher -> LaunchServices | `/usr/bin/open` activates the app bundle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --------- | -------- | --------- | ----------- | --------------- |
| T-01-02-S | Spoofing | `bin/make-an-issue` app target | mitigate | Resolve the app bundle relative to the launcher install root and require `.build/MakeAnIssue.app` to exist before calling `/usr/bin/open`. |
| T-01-02-T | Tampering | `launch-request.json` | mitigate | Store JSON with only `cwd` and `createdAtUnixSeconds`, decode strictly, test the exact shell-emitted fixture, clear malformed requests, and never execute values from the file. |
| T-01-02-R | Repudiation | Launch handoff | accept | Phase 1 does not create remote or privileged side effects; `createdAt` is enough local traceability for the latest handoff. |
| T-01-02-I | Information Disclosure | Application Support request file | mitigate | Store only the cwd needed for repo binding; do not include environment variables, shell arguments, user input text, or repository file contents. |
| T-01-02-D | Denial of Service | Malformed request file | mitigate | `consumeLatest()` must clear malformed JSON and return nil so a bad local file does not block app startup. |
| T-01-02-E | Elevation of Privilege | Launcher shell script | mitigate | Quote all paths, avoid `eval`, avoid sourcing shell files, default to `/usr/bin/open`, and permit `MAKE_AN_ISSUE_OPEN_COMMAND` only as an absolute-path automated-test override invoked without shell evaluation. |

</threat_model>
