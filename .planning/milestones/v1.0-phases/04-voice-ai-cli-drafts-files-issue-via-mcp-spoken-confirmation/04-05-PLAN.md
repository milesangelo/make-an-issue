<objective>
Fix the false-failure bug surfaced in UAT Test 4: the app shows "Issue tool not granted — check CLI
Command config" even though a real GitHub issue was created. Root cause (diagnosed in 04-UAT.md):
`IssueResultParser.parse` throws `.permissionDenied` whenever `permission_denials` is non-empty, and
that gate runs BEFORE returning the already-extracted successful issue url. When `claude` reaches for
any tool outside the allowlist (`mcp__github__issue_write Read Grep Glob`) during repo investigation,
that unrelated denial masks the successful `issue_write`.

Purpose: a successful `issue_write` url is proof the issue was filed; an unrelated denied tool must
not override it. The denial gate exists to prevent false *success* on a *failed* filing — that intent
is fully preserved: it must still fire when no issue url was found.

Output: reordered parse logic + a regression test pinning the corrected behavior. Surgical, two files.
</objective>
