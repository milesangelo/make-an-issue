## Overview

v1.1 removed the serial-filing bottleneck and added direct control over background filings and
drafting instructions. It preserved the v1.0 voice-to-issue pipeline while introducing concurrent
jobs, full-process-tree cancellation, an AppKit status-item shell, a live recording indicator,
editable instructions with an enforced app-owned contract, and a visible per-job status surface.

## Phases

### Phase 5: Concurrent Filing Jobs Model

**Goal:** Return capture to idle as soon as transcription completes and run multiple filings
concurrently, with each job retaining its own transcript, repository, state, and spoken outcome.

**Depends on:** Phase 4 (v1.0 filing pipeline)
**Requirements:** CONCUR-01, CONCUR-02, CONCUR-03
**Plans:** 2/2 complete

- [x] 05-01 — Introduce `FilingJob`, concurrent task-per-job execution, and deferred announcements.
- [x] 05-02 — Lock concurrency, isolation, retention, and announcement behavior with tests.

**Outcome:** Capture no longer waits for issue filing; multiple isolated filings can run at once,
and each completion is announced independently.

### Phase 6: Cancellation / Stop Control

**Depends on:** Phase 5

**Plans:** 4/4 complete

**Outcome:** Stop and quit operate on the full subprocess tree. UAT found a quit-time tempfile

## Milestone Summary

### Key Decisions

- Separate microphone capture state from independent filing-job state.
- Use one Swift task per filing and defer TTS while recording is active.
- Cancel the full spawned process group, with TERM followed by bounded KILL escalation.
- Own the status item and Settings window in AppKit for macOS 13–15 behavior.
- Keep drafting guidance editable but keep tool scope and issue-output requirements app-owned.
- Keep terminal job rows until explicit user dismissal; dismissal is not cancellation.

### Issues Resolved

- Serial filing no longer blocks the next push-to-talk capture.
- Individual filings can be stopped from the jobs list.
- Quit-time process and MCP-tempfile cleanup is deterministic.
- Recording state is visible without opening the popover, closing GitHub issue #2.
- The dead CLI Command field was removed.
- Failed filings retain a visible message and originating transcript.

### Issues Deferred

- Developer-ID signing and notarization for clean-machine distribution.
- Additional AI/MCP providers beyond the proven Claude + GitHub path.
- Review/edit-before-filing and advanced retry/queue behavior.

- One Phase 8 documentation-precision warning.

Nyquist coverage is partial for all five phases. Product verification still passed: 15/15
