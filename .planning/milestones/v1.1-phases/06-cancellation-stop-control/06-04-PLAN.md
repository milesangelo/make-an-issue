<objective>
Add the quit-time teardown so quitting the app while filings are in flight leaves no orphaned
processes and no leaked per-invocation MCP tempfiles (CANCEL-03 / D-04).

On `AppDelegate`, implement `applicationShouldTerminate(_:)`:
- Fast path (no `.filing` jobs): sweep `make-an-issue-mcp-*.json` tempfiles and return `.terminateNow`.
- Slow path (filings in flight): call `appState.cancelAll()` (immediate group SIGTERM via the 06-02
  bridge so docker `--rm` can clean up), return `.terminateLater`, and run a bounded `@MainActor`
  teardown Task that â€” inside a `defer { NSApp.reply(toApplicationShouldTerminate: true) }` â€”
  waits a 2s grace, calls `appState.forceKillAllProcessTrees()` (SIGKILL any survivors), and sweeps
  the tempfiles. Add a testable `sweepMCPTempFiles(in:)` glob delete scoped to the
  `make-an-issue-mcp-` prefix + `.json` suffix.

D-06 is accepted: on a real quit the process is exiting, so the cancel's spoken "filing cancelled"
realistically will not reach the speaker â€” status/log is the surface there; CANCEL-02's spoken
requirement is satisfied by the user-initiated cancel path (06-03). No extra work for that here.

Purpose: CANCEL-03 â€” clean shutdown (subprocess trees + tempfiles) with the SIGTERM-first ordering
D-04 requires, and a guaranteed terminate reply so quit never hangs.
Output: the quit hook + sweep, and a unit test proving prefix/suffix-scoped tempfile isolation and
the fast-path `.terminateNow`.
</objective>
