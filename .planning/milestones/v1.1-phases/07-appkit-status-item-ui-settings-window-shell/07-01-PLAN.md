<objective>
Replace the SwiftUI `MenuBarExtra` scene with a self-owned AppKit shell owned by `AppDelegate`:
an `NSStatusItem` whose button discriminates left vs right click, a transient `NSPopover` hosting
the existing `MenuView` (left-click), a programmatic `NSMenu` with exactly Settings…/Quit
(right-click, D-07), a self-owned `NSWindowController` Settings window hosting a new `SettingsView`
(D-03), and a single Combine sink that tints the menu-bar button red while
`captureState == .recording` (D-01/D-02, FEEDBACK-02). The `App.body` scene becomes a
`Settings { EmptyView() }` placeholder.

Purpose: deliver SETTINGS-01 (right-click Settings…/Quit + left-click popover, focusable Settings
window) and FEEDBACK-02 (live recording indicator) on a self-owned shell that behaves identically
across macOS 13/14/15 — avoiding the `MenuBarExtra`/`showSettingsWindow:`/`contentTintColor`
divergences (RESEARCH Pitfalls 7/8/9/B).

Output: new `SettingsView.swift`; extended `AppDelegate.swift` (shell + indicator, teardown
preserved verbatim); `MakeAnIssueApp.swift` scene swap.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user ↔ menu-bar UI | Local single-user interaction (click/keyboard). No network, IPC, or untrusted input crosses here. |
| Quit menu item → process teardown | Right-click "Quit" routes into the existing app-termination path that tears down in-flight subprocess trees. |

Phase 7 adds NO new input surfaces, NO new data stores, and NO new network or subprocess calls
(07-RESEARCH.md § Security Domain). No external packages are added this phase (Package Legitimacy
Audit: none), so there is no supply-chain install vector and no `T-07-SC` checkpoint is required.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | `showRightClickMenu` Quit item → `NSApplication.terminate(_:)` → `applicationShouldTerminate` | mitigate | Quit routes through the preserved Phase 6 teardown: `cancelAll()` SIGTERMs in-flight process groups, a 2s grace then `forceKillAllProcessTrees()` SIGKILLs survivors, and `sweepMCPTempFiles()` removes per-invocation MCP tempfiles. `applicationShouldTerminate` is preserved verbatim (Task 2) — do not alter. |
| T-07-02 | Spoofing | `showSettingsWindow` / `togglePopover` call `NSApp.activate(ignoringOtherApps: true)` | accept | Intentional and required for an LSUIElement app to give the Settings window / popover real keyboard focus (Pitfall 8/9). No user data is exposed in the empty Settings shell; single-user local app. |
</threat_model>
