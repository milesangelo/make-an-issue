<threat_model>

## Trust Boundaries

| Boundary | Description |
| -------- | ----------- |
| Launch cwd -> repo resolver | A filesystem path captured from the user shell becomes binding input. |
| Local filesystem -> app state | Directory markers determine the bound repository. |
| App state -> menu UI | The resolved repo path becomes visible in the menu. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --------- | -------- | --------- | ----------- | --------------- |
| T-01-03-S | Spoofing | `RepoBinding.resolve(from:)` | mitigate | Canonicalize cwd paths and bind only to an ancestor containing a `.git` directory or `.git` file; tests cover nested paths and spaces. |
| T-01-03-T | Tampering | `.git` marker detection | accept | Phase 1 trusts local user-writable repositories; no privileged or remote action occurs from binding alone. Later issue creation still runs from the resolved repo root. |
| T-01-03-R | Repudiation | Binding replacement | accept | Phase 1 stores only the current binding and status; no remote side effect or audit requirement exists for binding changes. |
| T-01-03-I | Information Disclosure | `MenuView` repo path display | mitigate | Display only the repo name and path required by LAUNCH-03; do not list files, branches, remotes, environment variables, or command output. |
| T-01-03-D | Denial of Service | Parent directory walk | mitigate | Stop traversal at filesystem root, return nil for non-repo paths, and keep resolution synchronous over ancestor checks only. |
| T-01-03-E | Elevation of Privilege | Repo resolver | mitigate | Do not execute `git` or any external command in this resolver; inspect filesystem markers only. |

</threat_model>
