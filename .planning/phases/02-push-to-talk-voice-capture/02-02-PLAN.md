<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Microphone → app (TCC) | The OS gates microphone access behind a user-granted TCC permission; the app must declare a usage string and request access. |
| App → filesystem (Application Support) | The app writes a recording file to a user-domain directory it constructs itself. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Information Disclosure | Microphone capture | mitigate | Recording only occurs while the shortcut is physically held (push-to-talk, D-03); no always-listening path. `NSMicrophoneUsageDescription` + explicit `requestRecordPermission` at startup means the OS gates access and the user sees the standard TCC dialog (Pitfall 2). |
| T-02-04 | Tampering | WAV output path | mitigate | The output path is built entirely from `FileManager.urls(for: .applicationSupportDirectory)` + the hardcoded literals `MakeAnIssue` and `latest.wav` — no user-controlled path components, so no path traversal (02-RESEARCH.md Security Domain V5). Recordings are kept out of the bound repo per D-06. |
| T-02-05 | Information Disclosure | latest.wav at rest | accept | A single local recording overwritten in place (D-07) under the user's own Application Support directory; standard macOS file permissions apply. Single-user local utility — no additional encryption-at-rest required for v1. |
| T-02-EoP | Elevation of Privilege | Recording without consent | mitigate | `AVAudioRecorder.record()` is never called before permission is requested; on denial the recorder fails into a no-op rather than silently capturing (Pitfall 2 / Pattern 5). |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this plan — the only SPM dependency (KeyboardShortcuts) is added and audited in Plan 02-01. No new dependencies introduced here. |
</threat_model>
