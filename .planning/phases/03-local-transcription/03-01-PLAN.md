    Capture output with TWO separate `Pipe` instances (one stdout, one stderr). Attach a `readabilityHandler` to each `fileHandleForReading` BEFORE `try process.run()`, appending `handle.availableData` to a per-pipe `Data` accumulator only when the chunk `isEmpty` is false (empty data is the EOF sentinel â€” Pitfall 3).     Wrap termination in `withCheckedContinuation`. In `process.terminationHandler`, first set both `readabilityHandler`s to nil, then guard a single `resumed` flag (`nonisolated(unsafe) var resumed = false` declared in the continuation closure) â€” check-then-set before resuming. Decode accumulated `Data` as UTF-8 (fall back to empty string on nil) and resume once with success (exit 0) or failed (non-zero exit, include stderr). Wrap `try process.run()` in do/catch; on a spawn error, guard the same `resumed` flag and resume with a failed/error outcome (use a sentinel exit code such as -1).

    Add the timeout as a sibling `Task` that does `try? await Task.sleep(for: timeout)`, then `guard !Task.isCancelled, !resumed else { return }`, sets `resumed = true`, calls `process.terminate()` (SIGTERM), nils both handlers, and resumes once with the timeout outcome.
