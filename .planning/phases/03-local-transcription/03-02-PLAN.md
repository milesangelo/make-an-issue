<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MenuView TextField → UserDefaults → spawned shell command | The user-typed ASR command is persisted and later run via `/bin/zsh -lc` (through CLIRunner). |
| WAV file path → `{wav}` substitution → shell command | An absolute filesystem path is injected into the command string before it reaches the shell. |
| subprocess stdout/stderr → transcript display | External-process output crosses back and is rendered as selectable text in the menu. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | `{wav}` path substitution in `Transcriber.prepare()` | mitigate | The injected `latest.wav` path is wrapped in POSIX single quotes with embedded single-quote escaping (close/literal/reopen) so spaces or metacharacters in the path cannot break out of the argument or trigger shell expansion (D-04, ASVS V5). Verified by `testWavSubstitutionQuoting`. |
| T-03-06 | Tampering | empty / malformed ASR command | mitigate | `Transcriber.prepare()` rejects an empty command (`emptyCommand`) and a command missing the `{wav}` token (`missingWavToken`) BEFORE any `Process` is spawned (D-03/D-05, ASVS V5 input validation). No silent append fallback. Verified by `testEmptyCommandShowsError` / `testMissingWavTokenError`. |
| T-03-07 | Tampering | user-controlled command string contents | accept | By design (D-02) the entire command is trusted user input — equivalent to what the user types in their own terminal. No sanitization is applied to the command body, which is correct per the PROJECT.md v1 boundary and RESEARCH Security Domain. Only the injected `{wav}` path is escaped. |
| T-03-08 | Spoofing/Tampering | transcript (untrusted process stdout) rendered in UI | mitigate | stdout is displayed in a SwiftUI `Text` view (D-09) — never parsed or executed as code; stderr (diagnostics) is kept separate and never merged into the transcript (D-08). No injection path from transcript content into app logic in this phase. |
| T-03-09 | Denial of Service | hung / runaway ASR process | mitigate | Inherited from CLIRunner's 120s timeout (Plan 01, D-12); on timeout `Transcriber` throws `asrTimedOut` and AppState resets `captureState` to `.idle` so the user is never stuck (D-11/D-12). Verified by `testTimeoutResetsState`. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this phase — Foundation + SwiftUI `@AppStorage` are built in (RESEARCH Package Legitimacy Audit: none). No legitimacy checkpoint required. |

No high-severity threat remains unmitigated.
</threat_model>
