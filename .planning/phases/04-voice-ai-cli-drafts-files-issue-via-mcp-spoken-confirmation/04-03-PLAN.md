<objective>
Wire the issue-filing pipeline into the app state machine and menu. Extend `CaptureState` with
`.filing`, add an `onRunIssueFiling` seam (mirroring the existing `onRunTranscription` seam), store
an `AVSpeechSynthesizer` on `AppState`, and after a transcript is captured automatically file the
issue and speak "created issue #NUMBER". Surface the `.filing` state and a persisted CLI Command
field in `MenuView`.

Purpose: This is the end-to-end wiring that connects Phase 3's transcript to the filed issue and
the spoken confirmation (the v1 happy path). It honors the LOCKED auto-file decision — the transition
from a captured transcript to filing is automatic, with no review/confirm step.

Output: extended `AppState.swift`, `MenuView.swift`, and `AppStateTests.swift`. The filing seam is
injected for tests so the state machine is fully unit-testable without a real `claude` call.
</objective>
