<objective>
Refactor the serial filing pipeline into a concurrent per-job model. Lift filing out of the
single `captureState` enum so capture returns to `.idle` the instant transcription completes,
spawn each filing as an independent `Task` tracked in a `@Published var jobs: [FilingJob]`
array, and route each job's spoken outcome through a defer-until-mic-idle announcement queue.

This is a refactor of the existing v1.0 pipeline (`beginFiling()` â†’ N concurrent
`spawnFilingJob()` calls), not a new user-facing surface. The jobs list UI is Phase 9;
cancellation mechanics are Phase 6. The only user-facing output added here is spoken
confirmations and failures.

Purpose: Deliver CONCUR-01 (idle immediately), CONCUR-02 (concurrent filings), CONCUR-03
(independent spoken outcomes) by generalizing the proven single-Task filing path to N.
Output: `FilingJob.swift` (new), refactored `AppState.swift`, `MenuView.swift` compilation
fix, and the four breaking tests rewritten so `swift test` is green.
</objective>
