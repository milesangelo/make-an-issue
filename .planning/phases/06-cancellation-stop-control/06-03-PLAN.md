<objective>
Wire the cancel mechanism into the jobs model so a user-initiated cancel (and the quit-path
internal API) drives the CLIRunner bridge from 06-02 and produces the cancelled outcome.

On `AppState`:
1. Extend the `onRunIssueFiling` seam with an `onProcessStarted` callback and have `spawnFilingJob`
   inject a closure that stores each job's process group id (hopped to `@MainActor`) into
   `FilingJob.processGroupID`.
2. Add a `catch is CancellationError` arm to `spawnFilingJob` that sets the job `.cancelled` and
   speaks `"filing cancelled"` through `announce()` (defer-until-mic-idle, D-05) — retaining the job
   in `jobs[]` (D-02/D-03), filing no issue.
3. Add `cancel(jobID:)` (cancel one in-flight job's Task), `cancelAll()` (cancel every in-flight
   Task — the quit-path API), and `forceKillAllProcessTrees()` (quit-time group SIGKILL of any
   still-filing job's stored pgid).

Then update the existing `AppStateTests` to the new seam arity and flesh out the four cancel
scaffolds from 06-01.

Purpose: CANCEL-01 (cancel trigger that reaps the tree) and CANCEL-02 (cancelled outcome: spoken +
state, no issue), plus the `cancelAll()`/`forceKillAllProcessTrees()` quit-path APIs CANCEL-03 (06-04)
consumes. SC-4 (no hung job, single-resume) preserved.

behavior.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-initiated cancel / quit-path API (`cancel(jobID:)`, `cancelAll()`, `forceKillAllProcessTrees()`) → in-flight `FilingJob` Tasks and their process groups | Internal API keyed on a typed `jobID: UUID`; the only untrusted-ish input is the stored `processGroupID`, which must be guarded `> 0` before a group SIGKILL. |
| CLIRunner's (non-main) onSpawn callback → `@MainActor` AppState `jobs` mutation | The pgid callback fires off the main actor; storing it into `@Published jobs` must hop to `@MainActor` to avoid a data race. |
