<objective>
Resolve FINDING-06 (SETTINGS-05): remove the orphaned, dead "CLI Command" field from the
menu-bar popover and delete its backing `AppState.cliCommandKey`, then add the new
`AppState.instructionsKey` UserDefaults key constant that Plans 02 and 03 build on.

Per D-01, the "CLI Command" field is confirmed dead code (written to UserDefaults, never read
back, never reaches the runner) and a false affordance (the enforced flags are claude-shaped), so
it is removed entirely — not relocated or wired. Per D-01a, the now-empty `DisclosureGroup` and its
introducing `Divider()` plus the orphaned `isSettingsExpanded` state are removed too. Per D-05 the
replacement persistence key `instructionsKey` is added following the exact `cliCommandKey` template.

Purpose: eliminate the false affordance (SETTINGS-04 SC4 / SETTINGS-05) and lay the persistence-key
foundation for the editable instructions field.
Output: a popover with no settings disclosure, `AppState` without `cliCommandKey` and with
`instructionsKey`, a green build, and green existing tests.
</objective>
