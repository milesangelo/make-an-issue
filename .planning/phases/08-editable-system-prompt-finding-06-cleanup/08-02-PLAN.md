<objective>
Deliver the SETTINGS-04 core guarantee: no user edit to the drafting instructions can remove or
weaken the app-owned enforced contract. Extract the "Issue URL on the last line" instruction out of
the editable prompt body into an app-owned `IssueFilingRunner.enforcedTrailer` (D-03), add the single
canonical `IssueFilingConfig.defaultInstructions` constant (D-06), restructure `buildPrompt` into
`{app framing + transcript} + {user guidance} + {app-owned file-it directive} + {enforced trailer}`
(D-02), fall back to the default when the guidance is blank (D-08), and thread the persisted
instructions through `file()` read fresh per filing invocation (SETTINGS-02 backend, D-02
concurrency safeguard). The `--allowedTools` flag stays where it already is — appended by
`assembleCommand` outside the prompt — and is NOT touched.

Purpose: make the enforced tool-scope + issue-URL contract architecturally unbreakable by user edits.
Output: `defaultInstructions`, `enforcedTrailer`, a restructured pure `buildPrompt`, an
instructions-threaded `file()`, a fresh-per-invocation read in AppState's default filing closure, and
unit tests proving the trailer + tool scope survive arbitrary and blank instructions.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-editable instructions → prompt assembly (`buildPrompt`) | Untrusted free-text guidance is interpolated into the `-p` prompt body. |
| prompt assembly → AI CLI subprocess flags (`assembleCommand`) | The scoped `--allowedTools` grant is set as a command-line flag, structurally separate from the prompt body. |
| persisted UserDefaults(instructionsKey) → per-filing read | The persisted guidance is read fresh at each filing spawn and captured by value. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | User guidance text attempting prompt-injection to suppress the URL trailer or redefine the file-it directive | mitigate | Architectural (D-02/D-03): `enforcedTrailer` and the `method=create` directive are appended by the app AFTER the editable guidance, outside the editable value. Verified by a unit test asserting `buildPrompt(instructions: "<injection text>")` still ends with `enforcedTrailer`. |
| T-08-02 | Elevation of Privilege | User guidance attempting to escalate tool scope (e.g. request broader tool access) | mitigate | `--allowedTools` is a command-line flag built by `assembleCommand` from `config.allowedToolsArgument`; the editable field only reaches the `-p` prompt body, never the flags. `assembleCommand` is not modified. Verified by a test asserting the command still carries the scoped grant when the prompt is built from arbitrary instructions. |
| T-08-03 | Denial of Service | Blank/whitespace guidance producing a degraded/empty prompt | mitigate | D-08 blank-fallback substitutes `IssueFilingConfig.defaultInstructions` at build time. Verified by `buildPrompt(instructions: "")` and whitespace-only tests. |
| T-08-08 | Information Disclosure | Model output containing a spurious GitHub issues URL that the prose parser's `firstMatch` mis-picks (STATE.md residual concern) | accept | Out of scope per CONTEXT D-02 — `IssueResultParser` is explicitly unchanged this phase. The `enforcedTrailer` places the canonical `Issue URL:` on the LAST line, and the editable field is guidance-only (never parsed as output). Residual model-output ambiguity is a pre-existing, accepted parser behavior; no parser change is made here. |
| T-08-09 | Tampering | Concurrent filings reading stale/shared instructions state | mitigate | Instructions are read fresh from UserDefaults inside the per-invocation closure and captured by value (no `self`-cached/`@Published` state), preserving `spawnFilingJob`'s existing concurrent-isolation guarantee. |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No package installs this phase — `Package.swift` unchanged; no new dependency added. No `[ASSUMED]`/`[SUS]` packages. |
</threat_model>
