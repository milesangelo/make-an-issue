<objective>
Build the small, fully-testable logic surface that Phase 9's jobs-list UI (Plan 09-02) renders and drives — with zero changes to the already-built/tested filing pipeline.

This plan delivers three things, all automatable with the existing XCTest infra (no ViewInspector/SnapshotTesting exists — verified in Package.swift):
1. Two new pure `@MainActor AppState` array mutations — `dismiss(jobID:)` and `clearFinished()` — that remove terminal jobs only, never cancel or drop in-flight work (D-06, D-05, RESIL-01).
2. Expose the currently-`private static` (and dead) `AppState.message(for: IssueFilingError)` to the same target so the failed-row view can render it — do NOT re-author the strings (D-09, RESIL-01).
3. A pure `JobRowStyle` mapper (state→icon, state→color) plus an `openableIssueURL` https-scheme guard, so per-state styling (JOBS-01, D-01) and the done-row URL-open safety check (D-10) are both unit-testable rather than trapped in an un-inspectable view body.

Purpose: These are the only non-view additions this phase needs. Pulling per-state styling and the URL guard into pure functions is the ONLY way to get automated coverage in a project with no rendered-view test infra — the view composition in Plan 09-02 is then a thin, manual-UAT-only shell.

Output: `dismiss(jobID:)`, `clearFinished()`, an accessible `message(for:)`, and `JobRowStyle` — each backed by a new unit test.
</objective>
