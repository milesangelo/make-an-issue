# Release app signing

## Goal

Produce a release-mode MakeAnIssue.app that has all embedded whisper Mach-O files signed before the completed outer bundle is sealed, with a reusable signature verifier.

## Tasks

1. Update `scripts/build-app.sh` to build Release, inject the bundle version from a simple release override, sign nested binaries first and the completed app last, then invoke verification.
   - Verify: shell syntax and a release build produce a strict-valid app signature.
2. Add a signing verifier script that runs strict code-signature validation and documents why Gatekeeper assessment is intentionally not included for ad-hoc signatures.
   - Verify: the verifier exits successfully for the assembled app.
3. Reconcile the project agent-memory convention, preserving the existing Claude guidance in `AGENTS.md` and restoring `CLAUDE.md` as its compatibility symlink.
   - Verify: `fm-ensure-agents-md.sh .` reports the convention is satisfied.

## Constraints

- Keep this limited to build-script behavior and build-adjacent comments/docs.
- Do not modify `README.md`.
- Developer ID signing and notarization remain out of scope.
